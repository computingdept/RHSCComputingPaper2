Computational Thinking, Algorithms, and Programming

2.1 Algorithms ADA

An algorithm is a step-by-step set of instructions to solve a problem, like a recipe.

Key concepts: Abstraction (focus on important details), Decomposition (break into smaller steps), Algorithmic Thinking (step-by-step problem-solving).

2.1 Searching LaB

Key concepts: Linear (Look at every item from the START until you find it.), Binary(Check the MIDDLE, then search LEFT or RIGHT) .

2.1 Sorting BMI

Key concepts: Bubble (Compare neighbors, swap if needed, repeat until sorted), Merge (Divide list into halves, sort, then merge together) and Insertion (Take one item, insert it in order, repeat.) .

2.2 Programming Fundamentals s SIS

Programming involves using a high level language ,english like, to give instructions to a computer a line at a time - SEQUENCE

Important elements include decisions (SELECTION) and variables (store information), data types (text, numbers, true/false), and ITERATION (repeat actions).

2.3 Producing Robust Programs

Good programs are reliable and secure. Techniques include validation (check input is correct) and authentication (password protection).

Errors can be syntax errors (typos) or logic errors (wrong results).

2.4 Boolean Logic

Computers use logic gates to make decisions: AND, OR, and NOT.

Example: In an AND gate, both inputs must be 1 to get an output of 1.

2.5 Programming Languages & IDEs

Computers understand different types of languages: high-level (Python) and low-level (Assembly).

Programs are translated using a compiler (fast) or interpreter (easy to debug).
