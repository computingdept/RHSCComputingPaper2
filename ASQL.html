SQL Query Result Trace Table Challenges

Each challenge presents a unique, high school–themed database table (with no more than 5 records) and a SQL query. Fill in the result table with the rows returned by the query (or the final state of the table). Some queries use the * wildcard (e.g. SELECT *), so be sure to check the SQL statement carefully.

Newly added (Q11–Q20): Additional focus on INSERT, DELETE, DROP, JOIN, LIKE, and wildcards.

Q1: Race Results – Students in YearGroup 11

Scenario: The table RaceResults records 100m race results. Show the StudentID and TeamName for all students in year group 11.

Database Table: RaceResults

StudentID YearGroup TeamName Time
11GC1 11 Valiants 20.3
10VE1 10 Super-Team 19.7
10SM1 10 Super-Team 19.2
11JP2 11 Champions 19.65

SQL Statement:

01 SELECT StudentID, TeamName
02 FROM RaceResults
03 WHERE YearGroup = 11

Result Table: (Fill in the rows returned by the query)

StudentID TeamName

Q14: JOIN Students and ExamScores

Scenario: We have two tables: Students and ExamScores. Show the StudentID, Name, and Score for matching rows.

Table: Students

StudentID Name
10AB1 John Adams
10CD2 Mary Clark
11EF1 Susan Evans

Table: ExamScores

StudentID Score
10AB1 88
11EF1 92

SQL Statement:

01 SELECT s.StudentID, s.Name, e.Score
02 FROM Students s
03 JOIN ExamScores e
04   ON s.StudentID = e.StudentID;

Result Table:

StudentID Name Score

Q15: JOIN with a WHERE Clause
